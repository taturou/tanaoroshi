# 開発計画 (PLAN.md)

## プロジェクト概要
スーパーマーケット向けの棚卸し（在庫カウント）アプリ。スマートフォンのカメラで商品のバーコード（JANコード）を読み取り、商品名と数量を記録してCSV出力する。GitHub Pagesでホスティングし、スマートフォンから手軽に利用できるPWA（Progressive Web App）として構築する。
**完全なサーバーレス（クライアントサイドのみ）で動作し、すべてのデータはスマートフォンのブラウザ内（Local Storage等）に保存する。**

## 技術スタック
- **フロントエンド:** React (TypeScript)
- **ビルドツール:** Vite
- **スタイリング:** Vanilla CSS (モダン・シンプル設計)
- **データ保存:** Local Storage / IndexedDB (ブラウザ内蔵ストレージ)
- **ホスティング:** GitHub Pages
- **バーコードスキャン:** `html5-qrcode` または `react-qr-reader` （実装時に選定）

## コア機能要件
1. **バーコード（JANコード）読み取り機能**
   - スマホのカメラを起動し、リアルタイムでバーコードをスキャンする。
2. **商品名の自動取得機能**
   - スキャンしたJANコードを元に、外部API（Yahoo!ショッピングAPI等を想定）へクライアントから直接リクエストを送り、商品名を自動取得する。
   - *※ 開発時はモックデータを使用。APIキー（Client ID）は設定画面等からユーザーが入力し、端末のローカルに保存する。*
3. **商品名の手入力（フォールバック）機能**
   - APIから商品名が取得できなかった場合や、未登録の商品の場合は、手動で商品名を入力できるようにする。
4. **数量入力・リスト管理機能**
   - 商品の読み取り（または入力）後、棚にある商品の数量をテンキー等で入力する。
   - スキャンしたデータ（JANコード、商品名、数量）のリストを画面上に一覧表示し、編集や削除ができるようにする。
   - **データはすべて端末内に永続化され、アプリを閉じても消えないようにする。**
5. **CSVダウンロード機能**
   - 端末に保存されている全データをCSV形式で出力し、ダウンロードする。
6. **PWA対応**
   - スマホのホーム画面に追加して、オフライン状態でもアプリを起動できるようにする（API通信はオンライン時のみ）。

## 画面構成（予定）
- **スキャン＆入力画面:** カメラプレビュー、スキャン結果の表示、数量入力フォーム。
- **リスト画面:** 記録した全データの一覧、個別データの編集・削除、CSVエクスポートボタン。
- **設定画面:** APIキーの入力・ローカル保存、記録データの全クリアなど。

## 開発ステップ
1. **フェーズ1: 基盤構築**
   - ViteによるReact(TS)プロジェクト作成。
   - GitHub Pagesへのデプロイワークフロー設定。
2. **フェーズ2: コア機能の実装**
   - カメラスキャン機能の実装。
   - スキャンしたデータ（JANコード）と数量をLocal Storageに保存する状態管理の実装。
   - CSV出力機能の作成。
3. **フェーズ3: API連携とPWA化**
   - 商品名取得APIの連携と手入力フォールバック機能の実装。
   - manifest.json と Service Worker の追加によるPWA対応（オフライン起動のサポート）。
4. **フェーズ4: デザインの仕上げ・テスト**
   - スマホ向けに最適化されたモダン・シンプルなUIの実装。
   - 実機（スマホ）を想定した動作確認。
